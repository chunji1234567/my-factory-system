# Project Plan

## 阶段 1：后端基础 (完成)
- 建立 Django/DRF 项目、核心模型、信号、财务逻辑。
- 实现权限控制和 API（产品、订单、库存、财务）。
- 编写单元测试 (core/business) 和 API 集成测试。

## 阶段 2：前端基础 (进行中)
- 创建 React + Vite + Tailwind + TS 前端。
- 登录/鉴权、导航、模块化 UI。
- 接入真实接口：产品、分类、销售订单、库存调整、API 测试面板。

## 阶段 3：接口全接入
- 采购、收货、发货、财务汇总、流水等模块全部接线。
- 实现角色可见性控制，隐藏未经授权的面板。
- 完成 API 测试页面所有操作按钮、批量操作工具。

## 阶段 4：企业微信 & 部署
- 设计企业微信登录/入口。
- 部署后端与前端（遵循 `rules/deployment-rules.md`）。
- 编写运维/监控文档。

## 风险与TODO
- 需新增合作伙伴 CRUD API（当前仅 Admin）。
- 前端需根据真实 API 结构进一步优化类型与错误处理。

- 库存中心改为分类视图，新增批量出入库工具和分类/产品创建弹窗。
- API 测试面板提供销售单、发货记录等真实操作按钮。
