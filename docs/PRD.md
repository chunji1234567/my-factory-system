# My Factory System PRD

## 1. 产品愿景
为一家多品类工厂提供一体化运营平台，覆盖产品主数据、销售与采购流程、库存管理、财务往来以及发货跟踪。系统需支持 PC 与移动端访问，并可对接企业微信。

## 2. 角色 
- **Manager**：全权限，负责创建/审核订单、查看财务数据。
- **Warehouse**：管理库存、收货、库存调整，种类和产品的增删改，可更新修改采购订单的状态和添加采购订单的收获记录无权查看金额。
- **Shipper**：记录发货，可更新修改销售订单的状态，添加订单下的发货日志，无权查看金额

## 3. 功能列表
### 3.1 产品与库存
- 管理产品与分类（经理可直接通过 API/前端表单创建）。
- 库存中心按分类展示原材料，可筛选、搜索、批量出入库，支持生产领料与手动增减库存。
- 批量库存操作（多物料一键出入库）减少重复录入。
- 自产件图库：专门展示自制外壳，提供图片、型号、编号、库存的卡片视图，并支持在图库内直接入库/出库与新建产品（含图片上传）。

### 3.2 销售
- 创建销售订单（含明细、分批发货）。
- 记录发货日志，自动更新订单状态。
- 查看销售历史、过滤/分页/排序。

### 3.3 采购
- 创建采购订单、收货记录，自动调整库存。
- 采购列表支持状态、供应商、日期筛选。
- 列表行点击可展开显示订单明细与事件（与销售列表交互保持一致），方便仓库快速查看收货记录。
- 收货中心同时提供桌面表格与移动端卡片视图，移动端默认仅展示未完成订单并支持状态快捷筛选。

### 3.4 财务
- 财务流水记录（应收/应付），同步合作伙伴余额。
- 财务汇总接口：按客户/供应商展示未结金额、订单数、转账历史。


## 4. 非功能需求
- **安全**：所有接口需 JWT 认证，并根据角色隐藏敏感字段。
- **性能**：主要列表支持分页、过滤、排序。
- **可维护性**：严格遵守 `rules/` 中的编码规范（SOLID、DRY、KISS、可读性）。

## 5. 里程碑
1. 后端初版（模型、API、权限、测试）
2. 前端基础框架（登录、导航、mock UI）
3. 前端对接核心接口（产品/订单/财务）
4. 企业微信对接与部署
