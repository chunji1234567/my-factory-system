# My Factory System PRD

## 1. 产品愿景
为多品类工厂提供一体化的前后端运营平台，让经理、仓库和发货团队在浏览器内即可完成主数据维护、订单执行、库存收发、财务对账以及异常记录。前端以 PC Web 为主，兼顾平板/移动端的核心流程，通过统一的 UI 组件和角色控制让跨部门协同更高效，后续可无缝接入企业微信等外部系统。

## 2. 角色
- **Manager（经理）**：拥有全部面板权限，可创建/编辑销售与采购订单、查看收发货详情、维护合作方与财务流水、导出报表。
- **Warehouse（仓库/产线）**：聚焦库存中心、收货中心与自产图库，可创建分类/产品、录入库存调整与收货日志，但不展示订单金额与财务信息。
- **Shipper（发货）**：进入发货控制面板以及销售订单事件记录，支持多批次发货、更新订单状态，隐藏金额类字段。

## 3. 功能模块

### 3.0 登录与导航
- 登录页提供用户名/密码表单并对接 `AuthContext`，反馈网络错误与“登录中”状态。
- 登录后顶部导航展示当前角色可访问的面板（源于 `panelConfig`），使用 `NavbarButton` 高亮当前页，active panel 同步到 URL `?panel=` 及 `localStorage` 方便刷新或分享。
- 每个面板渲染前通过对应的 `use*` hook 拉取远端数据（产品、分类、订单、合作方等），在 `loading`/`error` 态时给出提示或空状态。

### 3.1 库存中心（Inventory Panel）
- 支持按名称/内部编号搜索，按原材料分类筛选（若仅有单一分类会自动选中），结果按分类分组展示。
- “新建分类 / 新建产品”按钮触发 `CategoryForm`、`ProductForm` 弹窗，保证仓库可以在一处完成主数据维护。
- 列表可多选产品，逐个填写调整数量、选择操作类型（`MANUAL_IN`/`MANUAL_OUT`/`PRODUCE_IN`），批量调用库存调整 API 并提示校验信息。
- 勾选与数量输入的状态存储在组件内部，提交成功后自动清空并触发 `onRefreshProducts`。

### 3.2 销售管理（SalesOrdersPanel）
- 顶部使用 `FilterBar` 组合客户筛选（`PartnerSelect`/datalist 自动解析 #ID）与状态筛选，并提供“重置筛选”按钮。
- 表格展示客户、单号、状态、明细、金额、创建时间，可点击行展开 `OrderDetailsView` 查看明细行与事件时间线。
- 列表操作根据权限展示：`isManager` 可编辑/删除订单，`canCreateEvents` 可弹出事件表单（发货 / 退货 / 备注）并写入 order events。
- “新建/编辑销售单”弹窗由 `OrderItemsEditor` 驱动，支持选择分类/物料或自由输入客户产品名、同步客户偏好型号（`useCustomerPreferredProducts`）。删除包含发货记录的明细时会弹窗确认。
- 订单分页、展开、事件记录以及状态 badge 样式与采购/发货保持一致。

### 3.3 采购管理（PurchasePanel）
- 复用 `FilterBar + NavbarButton`，按供应商（只显示 `SUPPLIER/BOTH`）和状态筛选采购单。
- 表格列出供应商、单号、状态、金额、创建时间，行内展开 `OrderDetailsView` 查看明细/事件；加载态、错误态集中展示。
- 经理可新建、编辑、删除采购订单：表单支持分类->产品两级联动、状态选择、数量与价格校验，并提示输入有效供应商；保存后刷新列表。
- `canCreateEvents` 时可写采购事件（收货、退货、备注），用于补充仓库沟通记录；同时记录收到数量，用于收货中心比对。

### 3.4 发货控制（ShippingPanel）
- 面板拆为两个区域：左侧批量录入发货（可同时添加多条草稿，每条选择订单、明细、数量、物流单号，校验剩余可发数量），右侧为订单状态与发货日志。
- 顶部 FilterBar 支持客户/状态模糊筛选，自动解析客户 #ID；筛选结果驱动订单表格与发货日志列表共用的分页。
- 订单卡上可直接修改状态（带草稿状态和保存中提示）、记录事件，并查看每条明细的已发/待发数量。
- 发货日志模块加载 `useShippingLogs` 的历史记录，展示事件类型、创建时间、关联订单与明细，支持同一套客户筛选条件。

### 3.5 收货中心（WarehouseReceivingPanel）
- 主要面向仓库：FilterBar 仅暴露供应商筛选与“重置”，并按照状态优先级（已下单→部分→全部）排序展示待收货采购单。
- 列表支持桌面表格 + 可折叠卡片视图，点击采购明细可打开收货弹窗，输入本次数量与备注，系统会校验剩余可收数量并调用 `api.createReceivingLog`。
- 列表顶部提供状态快捷标签（已下单 / 部分入库），用于移动端快速筛选；收货完成后触发 `onRefresh` 更新状态与剩余数量。

### 3.6 自产图库（SelfMadeGalleryPanel）
- 只展示分类为 `SELF_MADE` 的产品，提供型号/编号搜索、分类筛选与分页（12 张卡片 / 页）。
- 卡片包含产品图、基础信息、现有库存与低库存提示；卡片内部按钮可直接进行入库/出库调整，填写数量和备注后调用库存调整 API。
- “新建产品”按钮复用 `ProductForm`，方便在图库内直接扩充新品；支持图片地址自动拼接 API base 兼容 OSS。

### 3.7 合作方中心（PartnerManagementPanel）
- 上方为创建合作方表单（名称、类型），提交后刷新列表并提示成功/失败。
- 列表可按类型筛选并按余额排序，点击任一合作方打开右侧详情抽屉，切换“订单 / 流水 / 台账”三种视图。
- 详情内展示未结金额、订单/流水列表以及台账分页，支持按日期筛选并导出台账（含事务来源、借贷发生额等）。

### 3.8 财务流水（FinanceDetailPanel）
- FilterBar 覆盖合作方 datalist、起止日期，过滤 `useFinanceTransactions` 返回的记录。
- 经理可新建流水（收款/付款/调整）并选择合作方、金额、备注；支持在列表中编辑/删除已有流水，所有操作都有 loading / success / error 提示。
- 列表展示方向、金额、合作方、备注、创建时间，并与合作方详情联动。

### 3.9 共用交互与组件
- `FilterBar`、`NavbarButton`、`PartnerSelect`、`StatusBadge`、`OrderDetailsView`、`OrderItemsEditor`、`Pagination` 等组件在各面板之间复用，保证筛选、操作按钮、列表样式一致。
- `usePartnerSearch`、`buildPartnerSuggestions`、`resolvePartnerId` 提供统一的合作方输入体验，避免 ID/名称解析逻辑重复。
- 事件记录与订单明细统一结构，方便未来拓展“导出事件”或“通知”能力。

## 4. 非功能需求
- **安全**：所有前端数据请求均依赖登录后获取的 JWT，面板渲染前根据用户角色过滤；金额类字段仅对拥有权限的角色展示。
- **性能与可用性**：列表默认分页（30 条），筛选/分页会复用 hook 缓存；输入框提供 datalist 以减少无效请求，并在批量提交时串行调 API，失败时展示明确错误。
- **可维护性**：代码遵循 `rules/` 中的规范，所有面板共享 TS 类型、hooks 与 UI 组件，减少重复逻辑；复杂表单拆分为可复用的编辑器组件。
- **一致性与可用性**：所有表单都有明确的错误提示和禁用态；主要操作按钮遵循“深色=主操作、描边=辅助/重置”的视觉约定。

## 5. 数据与集成假设
- 前端通过 `api/client` 与后端 REST API 交互，`useProducts/useSalesOrders/...` Hooks 负责生命周期与刷新；本地不做离线存储。
- 订单状态字段沿用 `ORDERED/PRODUCING/SHIPPED/COMPLETED`（销售）与 `ORDERED/PARTIAL/RECEIVED`（采购），事件类型保持与后端枚举一致。
- 未来需预留企业微信或其他通知渠道，可在事件记录成功后触发 webhook；图库图片路径可能来自 OSS，已通过 `VITE_API_BASE_URL` 适配。

## 6. 里程碑
1. **基础能力**：完成登录、角色鉴权、导航与全局样式。
2. **主数据与库存**：上线分类/产品表单、库存中心的搜索与批量调整、自产图库核心视图。
3. **订单流程**：交付销售/采购面板的创建、编辑、事件体系，并与合作方、产品数据打通。
4. **执行面板**：实现发货控制与收货中心，支持发货/收货日志、状态同步。
5. **财务与对账**：上线合作方中心与财务流水、台账导出，并为后续企业微信/部署留出接口。
